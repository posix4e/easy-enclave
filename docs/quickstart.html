// QUICKSTART

Get your first attested service running in under 5 minutes. No TDX expertise required.

Prerequisites

  • Python 3.9+
  • A GitHub repository
  • A docker-compose.yml for your service
Don't have TDX hardware? That's fine. The GitHub Action handles deployment to our TDX hosts. You just need the SDK to connect.

Step 1: Install the SDK

pip install easyenclave

Step 2: Add the GitHub Action

Create a workflow file in your repo at .github/workflows/deploy.yml:

name: Deploy to TDX

on:
  workflow_dispatch:
    inputs:
      ssh:
        description: 'Enable SSH access (dev only)'
        type: boolean
        default: false

jobs:
  deploy:
    uses: anthropics/easyenclave/.github/workflows/pipeline-dev.yml@main
    with:
      ssh: ${{ inputs.ssh }}
    secrets: inherit

Step 3: Create Your docker-compose.yml

Define your service. EasyEnclave will deploy this inside a TDX VM:

version: '3.8'

services:
  app:
    image: your-image:latest
    ports:
      - "8080:8080"
    environment:
      - PUBLIC_VAR=${PUBLIC_VAR}
    # Private env vars are injected securely, never persisted

Step 4: Trigger Deployment

Run the workflow from GitHub Actions UI or CLI:

gh workflow run deploy.yml
What happens next:
  • Your docker-compose is bundled and sent to a TDX host
  • The agent generates a TDX quote (hardware attestation)
  • Attestation is published as a GitHub release
  • Your service is live and verifiable!

Step 5: Connect from Your Client

from easyenclave import connect

# Connect to your attested service
# This fetches the attestation from your GitHub releases
# and verifies the TDX quote automatically
client = connect("your-org/your-repo")

# Now use it like any HTTP client
response = client.get("/health")
print(response.json())

# Access your API
data = client.post("/api/process", json={"input": "data"})
print(data.json())

That's It!

You now have:

  • A service running on hardware-attested infrastructure
  • Cryptographic proof published to GitHub
  • Clients that can verify the attestation before connecting
Production Checklist:
  • Remove ssh: true for production deployments
  • Use pipeline-release.yml for sealed production VMs
  • Set up private environment variables as GitHub secrets
